orxDLLAPI orxOBJECT* orxFASTCALL orxObject_FindChild ( const orxOBJECT *  _pstObject,
const orxSTRING  _zPath 
)

Finds the child inside an object's frame hierarchy that matches the given path. See orxObject_SetOwner() and orxObject_SetParent() for a comparison of ownership and parenthood in Orx. Note: this function will filter out any camera or spawner and retrieve the child matching the provided path. Paths are composed by object names separated by '.'. A wildcard can be used * instead of a name to find children at any depth inside the hierarchy, using depth-first search. Lastly, C subscript syntax, '[N]', can be used to access the N+1th (indices are 0-based) object matching the path until there. For example:

1 orxObject_FindChild(pstObject, "Higher.Lower"); will find the first child named Lower of the first child named Higher of pstObject
2 orxObject_FindChild(pstObject, "Higher.*.Deep"); will find the first object named Deep at any depth (depth-first search) under the first child named Higher of pstObject
3 orxObject_FindChild(pstObject, "*.Other[2]"); will find the third object named Other at any depth under pstObject (depth-first search)
4 orxObject_FindChild(pstObject, "Higher.[1]"); will find the second child (no matter its name) of the first child named Higher of pstObject
Parameters
[in]_pstObjectConcerned object
[in]_zPathPath defining which object to find in the hierarchy (cf. notes above)
Returns
